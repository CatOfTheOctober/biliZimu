# Bilibili 字幕下载项目逻辑梳理

该项目是一个功能完善的 Bilibili 视频字幕提取工具，其核心在于**智能两阶段处理逻辑**，旨在通过最优的方式获取或生成高质量的视频字幕。以下是项目的整体主流程与模块逻辑：

## 1. 核心处理流程

### 第一阶段：API 获取（官方/AI字幕优先）
1. **输入处理**：接析用户输入的视频 URL 或直接提取 BVID 和 cid。
2. **鉴权初始化**：加载基于 `BBDown` 工具生成的 Cookie 文件（这对获取 AI 生成字幕至关重要，因为部分接口受限）。
3. **API 请求**：通过官方接口请求该视频的字幕列表。系统会优先寻找并匹配 B 站自动生成的 AI 字幕（如 `ai-zh`）或 UP 主上传的官方字幕。
4. **解析与下载**：如果成功拿到字幕的 URL，直接下载 JSON/XML 格式的字幕文件，并将其解析为规范的文本片段（起始时间、结束时间、内容）。

### 第二阶段：ASR 语音识别（无字幕时的兜底方案）
如果通过第一阶段未找到任何可用的官方或 AI 字幕，系统会自动降级执行本地端语音识别引擎：
1. **视频/音频下载**：调用内嵌或外部的 `BBDown` 工具将该 B 站视频或音轨下载到本地临时目录。
2. **音频提取**：使用 `FFmpeg` 提取出适合语音识别的音频流（通常是 16kHz 单声道 WAV）。
3. **本地 ASR 处理**：
   - **FunASR**：默认调用阿里开源的中文语音识别引擎（支持 INT8/ONNX 加速），对中文处理更优。
   - **Whisper**：可选调用 OpenAI 的 Whisper 引擎处理多语言需求。
4. **文本组装**：将 ASR 引擎生成的语音转写结果和时间轴整合为规范的字幕格式。

## 2. 输出与文件管理
- **多格式生成**：无论字幕来源于 API 还是 ASR，都会统一转化并输出为用户所需的格式（如 `SRT` 默认、`TXT`、`JSON`、`Markdown` 等）。
- **资源管理**：自动清理下载视频、音频以及识别过程中产生的临时文件。

## 3. 项目架构特征
- **解耦设计**：核心逻辑层 (`extractor.py`) 控制流程，功能模块拆分为 API (`bilibili_api.py`)、解析 (`subtitle_parser.py`)、下载 (`video_downloader.py`)、语音识别 (`asr_engine.py`) 等独立组件，便于扩展维护。
- **环境依赖**：无需重量级的 Python 环境即可完成通过 API 的字幕提取。如果需要开启第二阶段的 ASR，才会按需加载庞大的 AI 推理库（如 `funasr`）。
