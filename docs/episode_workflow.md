# 单期节目包制作规范

## News Card 字段模板

- `title`
  - 这条新闻在你视频里的显示标题。
- `direct_scope`
  - 只写原节目里的直接问题边界。
  - 例：`只跟踪柳州城投债案例的公开处置进展，不扩展到全国城投债系统风险。`
- `original_background`
  - 用 `2-4` 句话说明当时发生了什么、为什么重要、为什么进入节目。
- `host_quote`
  - 主持人关键原话摘录，必须回看原视频核对。
- `host_view_summary`
  - 用中性语言概括主持人主要评论，不扩大、不替他说更多。
- `timeline`
  - 默认 `5-10` 个节点。
  - 少于 `5` 个通常说明后续材料不足，超过 `10` 个通常说明范围失控。
- `current_status`
  - 截至录制当天的阶段性状态，不写成历史终局。
- `allowed_sources_used`
  - 这条新闻实际使用过的来源标签，如 `official`、`新华社`、`财新`。
- `evidence_status`
  - `sufficient`
  - `insufficient_public_info`
- `clip_candidates`
  - 原节目里可用于引用或截图的时间段。

## 关键节点筛选规则

只保留满足以下至少一项的节点：

1. 官方首次回应或首次公开披露。
2. 事件方向发生明显转折。
3. 节目中的一个核心担忧或观察，在公开材料中出现了明确对应。
4. 当前状态判断必须依赖这个节点，否则观众无法理解后续走向。

默认排除：

- 重复报道同一事实
- 只增加细节但不改变判断的报道
- 扩展到相邻议题、行业大盘或泛政策史的材料
- 无法溯源到官方或白名单主流媒体的材料

## 单期脚本提纲模板

### 开场

- 交代原节目日期和回看视角。
- 用一句话概括本期一共讲了哪几条新闻。

### 每条新闻固定四段

#### 1. 背景

- 当时发生了什么。
- 这条新闻为什么值得进入节目。

#### 2. 原节目观点

- 读或展示主持人关键原话。
- 用一句中性概述承接到后续回看。

#### 3. 后续节点

- 按时间顺序讲 `5-10` 个关键节点。
- 每个节点画面角落标来源简称。

#### 4. 当前现状

- 只总结到录制当天为止的阶段状态。
- 如果公开信息不充分，直接说“公开信息不足”。

### 结尾

- 总结这期节目放到今天回看，最值得注意的变化是什么。
- 不做“说对/说错”打分，只做事实并列和阶段判断。
