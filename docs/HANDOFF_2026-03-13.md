## 3. 当前代码行为

### 3.1 第二流程输出结构

当前 `EpisodeDraft.json` 以 `schema_version = 2.1` 输出。

主题层：

- `news_topics[]`
  - `topic_id`
  - `canonical_topic`
  - `tracking_scope`
  - `retrieval_keywords`
  - `host_overall_view_summary`
  - `segments`
  - `review_status`

时间段层：

- `segments[]`
  - `segment_id`
  - `start`
  - `end`
  - `segment_summary`
  - `retrieval_keywords`
  - `host_view_summary`
  - `quote_anchors`
  - `angle_type`
  - `segment_role`
  - `subscope_label`
  - `confidence`

### 3.2 当前模型路由

- `sentence_analysis`
  - 在 `auto` 模式下默认走启发式
  - 目的是避免整期句级请求拖慢 CPU / API
- `segment_extract`
  - `auto` 模式优先本地，失败后转远程
- `topic_merge`
  - `auto` 模式优先本地，失败后转远程

### 3.3 本地模型超时配置

`.env` / `.env.example` 现在支持：

- `EPISODE_DRAFT_LOCAL_TIMEOUT_SECONDS`
- `EPISODE_DRAFT_API_TIMEOUT_SECONDS`
- `EPISODE_DRAFT_SENTENCE_ANALYSIS_MAX_ITEMS`

默认建议值：

```text
EPISODE_DRAFT_LOCAL_TIMEOUT_SECONDS=180
EPISODE_DRAFT_API_TIMEOUT_SECONDS=60
EPISODE_DRAFT_SENTENCE_ANALYSIS_MAX_ITEMS=120
```
