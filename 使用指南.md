# B站视频字幕提取工具 - 使用指南

## 🚀 核心工具：`下载字幕.py`（强烈推荐）

这是项目目前**最稳定、功能最全**的使用方式。它解决了 B 站 WBI 签名风控（412 错误）以及 ASR 本地模型识别的各种环境适配问题。

### 1. 运行环境
确保你使用的是包含 `funasr` 等依赖的 Python 环境（如 `conda activate py311`），然后直接运行：
```bash
python 下载字幕.py
```

### 2. 核心优势
- **自动降级**：如果官方/AI 字幕获取失败（412 错误），会自动提示并引导进入视频下载+本地 ASR 识别流程。
- **环境隔离适配**：直接通过 Python 运行，避免了打包成 CLI 后可能出现的 conda 环境冲突。
- **支持本地模型库**：默认使用 `D:/Funasr_model` 目录下的加速模型，识别速度极快且不占 C 盘空间。

## 🛠️ 其他工具说明

### 1. 命令行工具 (`bilibili-extractor`)
> ⚠️ **警告**：由于 Windows 命令行环境隔离问题，CLI 工具可能无法正确调用 conda 环境中的 ASR 模型，且更易触发 412 错误。**非开发需求请勿使用。**

### 2. 开发者模式
通过导入 `TextExtractor` 类在代码中调用：
```python
from bilibili_extractor.core.config import Config
from bilibili_extractor.core.extractor import TextExtractor

config = Config(funasr_model_path="D:/Funasr_model")
extractor = TextExtractor(config)
result = extractor.extract("URL")
```

## 📁 存储结构
- **模型文件**：`D:/Funasr_model`
- **字幕输出**：`output/` 目录下会生成 `.txt` 和 `.srt` 两个文件。
- **临时目录**：`temp/` 下载过程中产生的临时音视频，任务完成后会自动清理。

1. **首次使用**：运行 `cd tools/BBDown && BBDown.exe --login` 获取 Cookie

3. **批量处理**：在交互式界面中连续输入多个视频 URL

## 💡 提示

- **推荐使用 `下载字幕.py`**：这是最稳定的方式，避免 WBI 签名问题
- **Cookie 有效期**：B 站 Cookie 通常有效期较长，但建议定期更新
- **速率限制**：默认 20 秒间隔，避免频繁请求
- **文件命名**：自动处理特殊字符，确保文件名合法
- **编码格式**：所有文件使用 UTF-8 编码，支持中文
